# The DDS Unplugin

The Unplugin provides all necessary code to create a website in the look & feel of www.bs.ch. It's based on
[Tailwind](https://tailwindcss.com).

## Purpose and idea

The purpose of this Unplugin is to get developers started quickly with developing frontend components.

Since the Unplugin offers all the CSS, the only thing missing is the markup. **This is intentional.** Since
frameworks, such as Vue, Svelte, React, Twig, or Blade, differ wildly from each other in terms of template syntax,
providing a standardised way of including components is next to impossible. We, therefore, settled on providing the CSS
code and all assets necessary to create components, but not the components themselves. The markup for the components
can be found in Storybook.

## Scope

The plugin offers all the CSS code and assets, but no HTML and no JS. Again, this is intentional. Any built-in logic could
potentially interfere with existing logic from your framework (such as click handlers, looping, etc.), which is best
handled in your framework.

This means that, while you technically do not need any additional CSS, you will need business logic and data. You do not
need to take care of recreating any components from scratch or install any additional CSS framework.

## Technology

The Unplugin is based on the [**Unplugin system offered by unjs**](https://unplugin.unjs.io).  Behind the scenes,
Unplugin offers a standardised way to build plugins for all kinds of build tools, such as Vite, Webpack, Rollup,
ESBuild, Rspack and more. **This makes the DDS fully framework agnostic, leaving all the choices up to you.**

The DDS uses Tailwind to create all styles necessary for the entire DDS. The base config, containing spacings,
typography settings, colors, etc., is provided by the Unplugin. Tailwind allows you to style individual components
different, too. For example, if you need to add extra padding and margin to a certain element on the page, you can do
so by adding `p-*` and `m-*` classes to it.

The Unplugin also offers assets in the form of icons. These are provided as components and can be styled, too. Please
consult the examples provided in the Unplugin Github repository for further details.

Storybook also offers a way to inspect the entire Tailwind configuration. For this, please open the "Tailwind Config Viewer".

## How to install and set up the Design System Unplugin

The following instructions are taken from the README.md file of the Unplugin.

### Installation and setup

This package has several dependencies and peer dependencies.

#### Package installation

```bash
npm i git+https://github.com/kanton-basel-stadt/designsystem.git
```

Also, you need to install all peer dependencies for your framework:

For Vite, install

 * `vite`

For Webpack, install

 * `webpack`
 * `css-loader`
 * `mini-css-extract-plugin`
 * `postcss-loader`
 * `style-loader`

For Nuxt, install

 * `@vue/compiler-sfc`
 * `@nuxt/kit`
 * `@nuxt/schema`

For Rollup, install

 * `rollup`
 * `rollup-plugin-postcss`

For ESBuild, install

 * `esbuild`

### How to use icons

This unplugin uses [unplugin-icons](https://www.npmjs.com/package/unplugin-icons) as a nested plugin. All options of
this plugin apply, but they are set to sensible defaults in order to work correctly with the supplied icon set.

**Please keep in mind: In the examples specified above, the correct icon compilers are already configured.**

Use the following code snippets to include the icon `pen`. All other icons can be found in `src/core/assets/symbols/`
of this repository.

The following examples are what is implemented in the current examples. Since unplugin-icons also offers support for
various frontend frameworks, the `compiler` option may need to be different from the default configuration. For that,
please consult the official [unplugin-icons](https://www.npmjs.com/package/unplugin-icons#configuration) documentation.

### How to use the CSS

This unplugin uses PostCSS and Tailwind, both preconfigured. So, any tree-shaking of any Tailwind classes will be handled.

To include the framework in your CSS, apply the top of your CSS:

```css
@import '@@kanton-basel-stadt/designsystem/assets/css/tailwind.css'

/* Apply custom CSS here */
```

Please keep in mind that any additional CSS should be kept within `@layer components {  /* ... */ }`, so Tailwind knows
what to do with it.

If you want to simply use Tailwind as is with no additional CSS, you can do so by applying the desired classes to your
HTML elements. For that, please consult the official [Tailwind documentation](https://tailwindcss.com/docs/installation),
Storybook and the Tailwind config of this repository. Please keep in mind that Storybook is currently not up to date with
the latest implementations, especially when it comes to icons. They may need some adjustment.
